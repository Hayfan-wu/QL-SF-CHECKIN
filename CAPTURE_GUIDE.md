# 顺丰速运 sfsyUrl 一键抓取工具

电脑端扫码登录一键抓取顺丰速运完整 URL 的工具。

## 功能特点

- 🎯 **智能识别** - 自动识别并捕获顺丰速运相关请求
- ⭐ **质量评分** - 自动评估URL质量，推荐最佳可用URL
- 📱 **双模式** - mitmproxy 完整模式 / 简单代理模式（免安装）
- 💾 **自动保存** - 找到可用URL后自动保存到文件
- 📊 **历史记录** - 保存捕获历史，方便回溯
- 🌈 **彩色输出** - 美观的终端彩色输出

## 快速开始

### 方式一：一键运行（推荐）

```bash
# 安装依赖
pip install mitmproxy

# 运行抓包工具
python capture_sfsy.py
```

### 方式二：免安装模式

不安装 mitmproxy 也可以运行，但只能捕获 HTTP 请求（不推荐，顺丰接口多为 HTTPS）：

```bash
python capture_sfsy.py
# 提示是否安装 mitmproxy 时选择 n
```

## 输出文件

### sfsyUrl.txt

找到的最佳 URL，可直接复制到青龙面板的 `sfsyUrl` 环境变量中。

### sfsy_history.json

历史捕获记录（JSON 格式），保留最近 50 条。

## URL 质量评分

脚本会自动评估每个 URL 的质量分数：

| 评分项 | 加分 |
|--------|------|
| mcs-mimp-web.sf-express.com 域名 | +50 |
| shareGiftReceiveRedirect 路径 | +100 |
| shareRedirect 路径 | +90 |
| 包含 memberId 参数 | +80 |
| 包含 _login_user_id_ | +85 |
| 包含 _login_mobile_ | +85 |
| 包含 sessionId | +70 |
| 包含 token | +60 |

**高质量 URL 标准**（满足任一即可）：
- 同时包含 `_login_user_id_` 和 `_login_mobile_`
- `shareGiftReceiveRedirect` 路径 + `memberId` 参数
- `sessionId` + `_login_user_id_`

## 常见问题

### Q: 为什么捕获不到 URL？

A: 请检查以下几点：
1. 手机和电脑是否在同一 WiFi 下
2. 代理配置是否正确（IP 和端口）
3. mitmproxy 证书是否正确安装并信任
4. 是否打开了顺丰小程序并进入积分页面
5. 尝试关闭手机 VPN 或其他代理

### Q: iOS 安装证书后还是无法访问？

A: iOS 需要额外开启证书信任：
1. 设置 → 通用 → 关于本机 → 证书信任设置
2. 找到 mitmproxy 证书，开启完全信任

### Q: 可以用其他抓包工具吗？

A: 可以，如 Charles、Fiddler、HttpCanary 等。本工具是为了一键自动提取 sfsyUrl，省去手动查找的麻烦。

### Q: 支持多账号同时抓包吗？

A: 支持。只需在同一手机上切换账号操作，脚本会自动捕获所有请求，按质量排序展示。

### Q: 端口被占用怎么办？

A: 修改脚本中的 `Config.PROXY_PORT` 为其他端口即可。

## 高级用法

### 修改代理端口

编辑 `capture_sfsy.py`，修改：

```python
class Config:
    PROXY_PORT = 8888  # 修改为其他端口
```

### 添加自定义关键词

编辑 `capture_sfsy.py`，在 `TARGET_KEYWORDS` 中添加：

```python
class Config:
    TARGET_KEYWORDS = [
        'shareGiftReceiveRedirect',
        '你的自定义关键词',
        # ...
    ]
```

### 直接使用 mitmproxy 命令

```bash
# 直接用 mitmdump 加载插件
mitmdump -p 8888 -s capture_sfsy.py
```

## 技术原理

1. **代理模式**：在本地启动一个 HTTP/HTTPS 代理服务器
2. **请求拦截**：拦截所有经过代理的请求
3. **智能匹配**：匹配顺丰域名和关键词，捕获相关 URL
4. **质量评估**：根据 URL 特征评估可用性
5. **自动保存**：找到高质量 URL 后自动保存到文件
